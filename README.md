# AI Engineering Interview Vault

A public-facing learning and interview-preparation vault for AI engineering, AI reliability, retrieval systems, structured outputs, evals, agents, observability, privacy, and production-grade system design.

This repository is not a passive notes dump.

It is designed to turn study sessions into durable artifacts:

- session files
- concept pages
- runnable snippets
- interview answers
- mistake patterns
- spaced repetition prompts
- consultancy offer translations

## Purpose

The goal is to build interview-ready and client-ready understanding.

For each concept, the standard is to explain:

1. definition
2. example
3. counterexample
4. production failure mode
5. implementation pattern
6. evaluation/debugging approach
7. client or business implication

## Public vs private learning artifacts

This repo is public-facing.

Raw weak answers, messy drafts, false confidence logs, and private reflections live under `.local/`.

`.local/` is intentionally ignored by Git.

Public files should be polished, useful, and professional.

## Main folders

- `weeks/` — chronological learning sessions tied to roadmap weeks
- `concepts/` — canonical current understanding, one concept per file
- `snippets/` — runnable or near-runnable code examples
- `interview-bank/` — spoken answers and interview drills
- `mistake-patterns/` — public-safe recurring confusion patterns
- `review-schedules/` — spaced repetition plans
- `offer-translation/` — links between technical learning and consultancy offers
- `prompts/` — reusable learning prompts
- `templates/` — markdown and code templates

## Learning rule

Weekly files answer:

> What did I learn, attempt, miss, correct, and need to retrieve later?

Concept files answer:

> What is my best current reusable explanation of this concept?

Snippets answer:

> What production pattern or failure prevention does this code demonstrate?

Interview files answer:

> How would I explain this clearly under pressure?
